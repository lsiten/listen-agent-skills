# Agent Skills 开放标准参考

本文档是Agent Skills开放标准的快速参考指南，供创建skill时参考。

## 标准概述

**Agent Skills** 是Anthropic于2025年12月18日正式发布的AI智能体能力扩展开放标准。

### 核心理念

1. **渐进式披露（Progressive Disclosure）**
   - 解决上下文窗口膨胀问题
   - 按需加载信息，而非一次性全部读取

2. **连接性与能力分离**
   - 与MCP（模型上下文协议）互补
   - MCP专注外部系统连接，Skills专注任务执行方法

3. **去中心化与简单性**
   - 无中心化注册表
   - 无包管理器
   - 技能即文件夹

## 标准文件结构

```
my-skill/                  # 技能文件夹
├── SKILL.md               # 核心文件（必需）
├── instructions/          # 详细指令（可选）
│   └── ...
├── scripts/               # 可执行脚本（可选）
│   └── ...
└── resources/             # 参考资源（可选）
    └── ...
```

### 命名规范

- **目录名**：小写字母(a-z)、数字(0-9)、连字符(-)
- 不能以连字符开头或结尾
- 最大长度：64字符

## SKILL.md 格式规范

### 元数据（YAML Front Matter）

```yaml
---
name: my-skill               # 必需，小写、连字符，≤64字符
description: 技能描述         # 必需，1-1024字符
version: 1.0.0               # 可选
author: 作者名                # 可选
tags: ["tag1", "tag2"]       # 可选
---
```

### 必需字段

| 字段 | 规范 | 作用 |
|------|------|------|
| `name` | 小写字母、数字、连字符，≤64字符 | 唯一标识技能 |
| `description` | 1-1024字符 | 让Agent判断技能相关性 |

### Markdown正文

元数据后跟随Markdown格式的指令文档，包含：

- 任务概述
- 执行步骤
- 使用示例
- 参数说明
- 等等

## 渐进式披露机制

| 级别 | 触发时机 | 加载内容 | 作用 |
|------|----------|----------|------|
| **L1: 元数据扫描** | Agent启动 | name和description | 发现可用技能 |
| **L2: 指令激活** | 任务匹配时 | 完整SKILL.md | 提供执行步骤 |
| **L3: 资源访问** | 执行过程中 | 脚本、文档等 | 提供详细资源 |

## 内容编写指南

### 角色定义

开头清晰定义AI角色：

```markdown
你是一个专门处理XXX的AI助手，具备YYY能力。
```

### 任务概述

简洁说明skill功能：

```markdown
## 任务概述

本技能用于{{功能描述}}，支持{{特性列表}}。
```

### 核心能力

使用emoji增强可读性：

```markdown
## 核心能力

- 🎯 **能力1** - 说明
- 📊 **能力2** - 说明
```

### 执行步骤

详细、可操作的步骤：

```markdown
## 执行步骤

### 第一步：环境准备

```bash
具体命令
```

### 第二步：执行任务

```bash
具体命令
```
```

### 参数说明

使用表格展示：

```markdown
## 参数说明

| 参数 | 类型 | 必需 | 默认值 | 描述 |
|------|------|------|--------|------|
| `--param` | string | 是 | - | 描述 |
```

真实可用的示例：

```markdown
## 使用示例

### 示例1：基础使用

可直接复制执行的命令
```
```

## 与MCP的关系

| 特性 | Agent Skills | MCP |
|------|--------------|-----|
| 核心作用 | 任务执行方法 | 外部系统连接 |
| 实现方式 | 文件系统结构 | HTTP/JSON-RPC |
| 关注点 | 能力封装复用 | 系统集成 |
| 协作 | 互补使用 | 互补使用 |

## 质量检查清单

创建skill后，确保满足以下条件：

### 规范检查
- [ ] 目录名符合命名规范
- [ ] SKILL.md存在且格式正确
- [ ] name字段符合规范
- [ ] description字段在1-1024字符

### 内容检查
- [ ] 包含角色定义
- [ ] 包含任务概述
- [ ] 执行步骤可操作
- [ ] 示例代码可运行

### 可用性检查
- [ ] 脚本可执行（如有）
- [ ] 依赖说明清晰
- [ ] 错误情况有处理
