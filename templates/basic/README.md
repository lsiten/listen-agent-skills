## 适用场景

本模板适用于**简单直接**的skill，例如：
- 调用单一API或Webhook
- 发送通知消息
- 执行简单的数据转换
- 配置驱动的自动化任务

**参考实例**: `wechat-work-notification`

## 模板特点

- 📝 **简洁明了** - 结构简单，专注于核心功能
- 🔗 **配置为主** - 通过配置项驱动，减少复杂逻辑
- 🎯 **即开即用** - 提供场景模板，可直接复制使用

## 目录结构

```
{{skill-name}}/
├── SKILL.md          # 技能定义文件（必须）
├── README.md         # 说明文档
└── scripts/          # 辅助脚本（可选）
    └── install_dependencies.sh
```

## 快速开始

1. 复制本模板目录到 `skills/{{your-skill-name}}/`
2. 编辑 `SKILL.md`，替换所有 `{{placeholder}}` 为实际内容
3. 编辑此 `README.md`，填写技能描述
4. 测试技能功能

## SKILL.md 结构说明

```markdown
---
name: skill-name          # 技能唯一标识
description: 功能描述      # 1-1024字符
version: 1.0.0
author: ""
tags: ["tag1", "tag2"]
---

# 技能名称 Skill
## 任务概述              # 简要说明功能
## 核心能力              # 2-3个核心能力
## 配置信息              # Webhook地址、API密钥等
## 执行步骤              # 简单的执行命令
## 常用场景模板          # 可直接复制使用的模板
## 使用示例              # 完整的使用示例
## 最佳实践              # 使用建议
## 响应处理              # 成功/错误响应格式
## 注意事项              # 重要提醒
```

## 何时选择此模板

✅ **适合**：
- 功能单一、逻辑简单的skill
- 主要通过API调用完成任务
- 不需要复杂的多步骤流程
- 配置驱动而非逻辑驱动

❌ **不适合**：
- 需要多步骤数据处理的skill（使用 `advanced` 模板）
- 需要多阶段工作流的skill（使用 `complex` 模板）
- 需要复杂的状态管理和流程控制

## 注意事项

- 保持SKILL.md简洁，避免过度设计
- 提供可直接复制使用的场景模板
- 确保配置信息说明清晰完整
- 使用示例应包含可直接执行的完整命令
