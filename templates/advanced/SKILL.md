## AI执行方式

### 第一步：理解任务

当用户提出任务时，你应该：

1. **理解需求** - 仔细分析用户的需求，明确任务目标和期望结果
2. **识别上下文** - 理解当前的工作环境、可用资源和约束条件
3. **分析问题** - 将复杂问题分解为可管理的子问题
4. **确定方法** - 基于你的知识和能力，确定最适合的解决方案

### 第二步：规划执行

在执行任务前，你应该：

1. **制定计划** - 根据任务需求，制定详细的执行计划
2. **识别依赖** - 确定需要的外部资源、工具或信息
3. **评估风险** - 识别潜在问题和风险，准备应对方案
4. **准备工具** - 如需要，准备必要的辅助工具（脚本、MCP等）

### 第三步：执行任务

在执行任务时，你应该：

1. **独立思考** - 运用你的知识和推理能力，独立分析和解决问题
2. **逐步推进** - 按照逻辑顺序，逐步完成任务
3. **使用工具** - 当需要确定性操作时，可以使用脚本等辅助工具，但不应过度依赖
4. **验证结果** - 检查执行结果，确保符合预期

**重要**：主要工作应该由你（AI）通过思考和推理来完成。脚本只是辅助工具，用于：
- 安装依赖（`./scripts/install_dependencies.sh`）- 仅在需要时使用
- 格式化输出 - 仅在需要特定格式时使用
- 确定性操作 - 仅在需要精确执行时使用

### 第四步：结果处理

完成任务后，你应该：

1. **总结结果** - 清晰总结完成的工作和取得的成果
2. **提供反馈** - 向用户说明结果，解释关键决策和发现
3. **记录经验** - 如有必要，记录本次任务的经验和教训

## 最佳实践

1. **独立思考优先**
   - 优先使用你的知识和推理能力，而不是依赖脚本
   - 将脚本视为辅助工具，不是主要执行方式
   - 在需要确定性操作时才使用脚本

2. **合理使用工具**
   - 使用MCP工具访问外部系统
   - 使用脚本处理确定性操作（如安装依赖、格式化输出）
   - 使用资源文档获取补充信息

3. **清晰沟通**
   - 与用户保持清晰沟通，解释你的思考过程
   - 说明为什么选择某种方法或工具
   - 提供详细的执行反馈

4. **持续优化**
   - 从每次任务中学习，不断改进执行方式
   - 评估结果，优化方案
   - 记录经验，提高效率
