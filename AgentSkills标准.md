# Skill的开放标准：Agent Skills详解

**Agent Skills**是Anthropic（Claude开发公司）于2025年12月18日正式发布的**AI智能体能力扩展开放标准**，官方规范见[agentskills.io](https://agentskills.io)。它本质上是一套轻量级、可移植的结构化格式，用于将特定任务的专业知识、工作流程和可执行逻辑封装为**可复用、可迭代的能力包**，让AI Agent能够以确定性方式交付专业任务。

---

### 一、核心定义与设计理念

**定义**：一个Skill就是一个包含`SKILL.md`文件的文件夹，内含完成特定任务所需的全部"知识"——元数据、指令文档和可选资源。

**核心设计理念**：
- **渐进式披露(Progressive Disclosure)**：解决上下文窗口膨胀问题，按需求加载信息，而非一次性全部读取
- **连接性与能力分离**：与MCP(模型上下文协议)互补，MCP专注外部系统连接，Skills专注任务执行方法
- **去中心化与简单性**：无中心化注册表、无包管理器、无依赖解析，技能即文件夹，可通过本地磁盘、Git仓库、网络共享或zip分发

---

### 二、标准文件结构与格式

#### 1. 基本目录结构
```
my-skill/                  # 技能文件夹（小写字母、数字、连字符，不首尾连字符）
├── SKILL.md               # 核心文件（必需）
├── instructions/          # 详细指令子文件夹（可选）
│   ├── step-by-step.md
│   └── best-practices.md
├── scripts/               # 可执行脚本（Python/Shell等，可选）
│   ├── helper.py
│   └── run-task.sh
└── resources/             # 参考资源（文档、模板等，可选）
    ├── template.docx
    └── policy.pdf
```


#### 2. SKILL.md核心格式
该文件是YAML元数据与Markdown指令的混合体：
```markdown
---
# YAML元数据（必需）
name: my-skill               # 技能名称（小写、连字符，最大64字符）
description: 完成XX任务的专业技能包  # 描述（1-1024字符）
version: 1.0.0               # 版本号（可选）
author: John Doe             # 作者（可选）
tags: ["data-analysis", "reporting"]  # 标签（可选）
---

# Markdown指令文档（必需）
## 任务概述
本技能用于自动生成月度销售报告...

## 执行步骤
1. 调用MCP连接销售数据库
2. 提取上月销售数据
3. 运行scripts/analyze.py处理数据
4. 使用resources/template.docx生成报告
```


#### 3. 元数据必需项与规范
| 字段 | 要求 | 作用 |
|------|------|------|
| `name` | 小写字母(a-z)、数字(0-9)、连字符，不首尾连字符，≤64字符 | 唯一标识技能 |
| `description` | 1-1024字符 | 让Agent判断技能相关性的关键信息 |

---

### 三、渐进式披露三级机制

| 级别 | 触发时机 | 加载内容 | 作用 |
|------|----------|----------|------|
| **Level 1: 元数据扫描** | Agent启动时 | 所有技能的`name`和`description` | 让Agent知道有哪些技能可用，何时该使用 |
| **Level 2: 指令激活** | 任务与技能匹配时 | `SKILL.md`完整内容（元数据+Markdown指令） | 提供执行任务的完整步骤和方法 |
| **Level 3: 资源访问** | 执行过程中按需调用 | 脚本、参考文档等外部资源 | 提供确定性执行代码和补充信息，上下文不受限 |

这种机制使Agent能够高效管理大量技能，只在需要时加载相关内容，大幅提升上下文窗口利用率。

---

### 四、关键特性与优势

1. **跨平台兼容性**：遵循开放标准，可在Claude、Cursor、GitHub Copilot、VS Code等支持该标准的Agent中使用，实现"一次构建，处处部署"
2. **可执行性**：可包含Python/Shell等脚本，实现确定性任务执行，超越静态prompt模板
3. **可扩展性**：通过子文件夹和外部资源支持复杂任务，上下文容量几乎无限制
4. **可维护性**：结构化格式便于版本控制和团队协作，可像代码一样迭代优化
5. **安全性**：支持沙箱执行、敏感数据脱敏和权限控制，降低执行风险

---

### 五、与MCP协议的关系

| 特性 | Agent Skills | MCP(模型上下文协议) |
|------|--------------|---------------------|
| **核心作用** | 提供任务执行方法和专业知识（"操作手册"） | 提供外部系统连接能力（"万能插头"） |
| **实现方式** | 文件系统为基础的结构化文件夹 | 基于HTTP/JSON-RPC的通信协议 |
| **关注点** | 能力封装与复用 | 系统集成与数据交换 |
| **发布状态** | 2025年12月18日开放标准 | 2025年12月9日捐赠给Linux基金会 |
| **协作关系** | 互补关系，常一起使用 | 互补关系，常一起使用 |
